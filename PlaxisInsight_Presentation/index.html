Problem Description

WHAT IS A
CONVERGENCE LOG?

  • Every PLAXIS phase writes convergence log files (data.convd.rr#, data.convf.rr#)
  • It records the complete solver history: steps, iterations, errors, plastic points, rescaling and the phase summary.

The Pain

  • Raw Fortran-formatted text
  • Thousands of lines, encrypted structure
  • Format varies by analysis type (coupled, dynamic, safety, …)
  • Fields depend on element types (interface, geogrid, anchor, …)
  • Comparing runs = scrolling two text files side by side
  • No plots, no trends, no KPIs

Who Suffers?

  • Most users don't know these files exist
  • Those aware find them overwhelming and avoid them
  • Primary consumers:
  • CCG team — Investigating calculation kernel
  • Support team — Troubleshooting user projects

Aim & Scope — Origin Story

  • I was investigating kernel options affecting convergence speed in projects with discontinuity elements
  • Metrics I used: CPU/wall time, step count, iteration count — but the log contains far more
  • Beyond the basics — hidden in the log:
  • Convergence error trends per iteration
  • Plastic point evolution across steps (7 families × 4 row types)
  • Rescaling events: frequency, reasons, scale factors
  • Steady-state regime checks, extrema with node coordinates
  • Step size evolution, arc-length factors, relative stiffness
  • The spark: comparing two runs with different options was nearly impossible without tooling.
  • PlaxisInsight — parse, visualise and compare convergence log files in a smart and efficient way.

Challenges — Log File Structure

PART 1: HEADER
Solver Settings

  • CONVERGENCE CRITERIA
  • Thermal tolerances
  • GW flow / coupled criteria
  •   Steady-state regime
  •   Unsaturated behaviour
  •   Boundary node status
  • Mechanical / deformation
  •   Force, soil, moment error
  •   Interface, beam, geogrid, anchor tolerances

PART 2: STEP BLOCKS
Solver History

  • Per-iteration data:
  • Step size (with time unit)
  • 9 convergence error fields
  • Plastic pts: 7 families × 4 rows
  • Rescaling: factor + reason
  • 6 known block types
  • Step summary:
  • Converged YES/NO, ΣMStage/time
  • Stiffness, extrema with locations

THE CHALLENGE

  • Field presence varies by:
  • Calculation type
  • 16 types — Plastic, Consolidation, Safety, Dynamics, Fully coupled, THM, GW flow, Heat flow, …
  • Element types in model
  • Interface, geogrid, anchor, plate, beam, flex. hinge
  • Solver decisions
  • Rescaling, arc-length, sub-steps, attempts

Methodology — Parsing Logic

  • Rule-based pipeline — 13 ordered rules, first match wins per line:
  • ① Start known block (Steady-state, Check convergence, Flow BC, Unsaturated, Freeze/thaw, Time step upscale)
  • ② Known-block mode — state machine per block type with sub-modes and sentinels
  • ③ Unknown-block mode — indent-based hierarchy for unrecognised patterns
  • ④ Step size — extraction with time unit + negative detection
  • ⑤ Total iterations  |  ⑥ Plastic points header  |  ⑦ Plastic points rows
  • ⑧ Converged attempt (YES/NO)  |  ⑨ Inaccurate soil elastic points
  • ⑩ Rescaling reason — forward-paired with next numeric factor
  • ⑪ Unknown block start  |  ⑫ Standalone YES/NO  |  ⑬ Generic trailing metric
  • Documented structures (in Notes/):
  • StepBlock_Structure.txt — complete field tree with format specs
  • CalcType_ConveCri_StepOpt_Rules.txt — criteria hierarchy for all 16 calc types
  • Phase_Summary_Structure.txt — all phase-end messages

Live Tracking — Real-Time Monitoring

  • Monitor a running PLAXIS calculation — parse the convergence log as it grows.
  • Two modes:
  • Companion server (live-server.ts on localhost:9876) — reliable reads of files held open by PLAXIS, supports changing watched directory at runtime
  • File System Access API (browser-only fallback) — no server needed
  • Capabilities:
  • Phase selector modal — scans for data.convd.rr# files, shows phase numbers & sizes
  • Auto-detects newly appearing phases during calculation
  • Per-phase stop / resume — freeze individual phases while tracking others
  • Inactivity detection — warning after 15 s of no file changes
  • Content-length based change detection (OS metadata unreliable while PLAXIS holds file handles)
  • 1.5 s polling interval — low overhead, real-time feel

Summary

PlaxisInsight transforms raw convergence logs into an interactive diagnostic tool.

  • ✓ Parse — all 16 PLAXIS calculation types automatically
  • ✓ Visualise — convergence errors, stiffness, plastic points, rescaling — interactive charts
  • ✓ Compare — multiple analyses with overlay charts and summary tables
  • ✓ Custom plots — any metric vs. any metric, step or iteration resolution
  • ✓ Inspect — every iteration — dynamic columns, tolerance colouring, fullscreen mode
  • ✓ Export / Import — lossless JSON round-trip for sharing and archival
  • ✓ Live tracking — real-time monitoring of running PLAXIS calculations

Built for the CCG and Support teams — designed for any PLAXIS power user.

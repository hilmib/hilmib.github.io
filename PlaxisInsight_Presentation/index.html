Problem Description

WHAT IS A
CONVERGENCE LOG?

  • Every PLAXIS phase writes convergence log files (data.convd.rr#, data.convf.rr#)
  • It records the complete solver history: steps, iterations, errors, plastic points, rescaling and the phase summary.

The Pain

  • Raw Fortran-formatted text
  • Thousands of lines, encrypted structure
  • Format varies by analysis type (coupled, dynamic, safety, …)
  • Fields depend on element types (interface, geogrid, anchor, …)
  • Comparing runs = scrolling two text files side by side
  • No plots, no trends, no KPIs

Who Suffers?

  • Most users don't know these files exist
  • Those aware find them overwhelming and avoid them
  • Primary consumers:
  • CCG team — Investigating calculation kernel
  • Support team — Troubleshooting user projects

Motivation — How It Started

  • I was investigating kernel options affecting convergence speed in projects with discontinuity elements
  • Metrics I used: CPU/wall time, step count, iteration count — but the log contains far more
  • Beyond the basics — hidden in the log:
  • Convergence error trends per iteration
  • Plastic point evolution across steps (7 families × 4 row types)
  • Rescaling events: frequency, reasons, scale factors
  • Steady-state regime checks, extrema with node coordinates
  • Step size evolution, arc-length factors, relative stiffness
  • The spark: comparing two runs with different options was nearly impossible without tooling.
  • PlaxisInsight — parse, visualise and compare convergence log files in a smart and efficient way.

Challenges — Log File Structure

PART 1:
Solver Settings

Convergence criteria

Thermal: heat flow error, temperature change error, heat storage variation tolerances

GW flow / coupled: GW flow error tolerance, plus sub-groups for steady-state regime (local flow error, pore pressure change, relative permeability change, degree-of-saturation change), unsaturated behaviour thresholds, and boundary node status iteration limits (seepage, drain, well, ponding nodes)

Mechanical / deformation: global force error, soil local error, moment error, soil accuracy (plastic & elastic point tolerances), interface/discontinuity tolerances, embedded beam interface error, structural element tolerances (plate, beam), geogrid tolerances, anchor tolerances

Step size control parameters

Max iterations per step, desired min/max iterations, max consecutive negative steps

Control flags: Arc-length control (Off/On/Auto), Updated mesh (Off/On/Auto)

PART 2:
Solver History

Per-iteration data

Step size: with time unit label (e.g. "in days", "in seconds")

Convergence errors: up to 9 fields (force, soil, interface, moment, embedded beam, geogrid, anchor, GW flow, heat flow)

Plastic points: 7 element families (Soil, Interface, Plate, Beam, Geogrid, Anchor, Flexible hinge) × up to 4 rows (Total, Inaccurate, Total tension, Inaccurate tension)

Inaccurate soil elastic points: standalone scalar

Rescaling events: scale factor + reason (7 known reasons)

6 known sub-blocks: Check convergence, Steady state regime, Flow boundary condition changes, Time step upscale validity, Unsaturated behaviour monitoring, Freezing/thawing heat storage change

Iteration variants: Attempt / sub-step / final stress update headers

Step summary

Converged: YES / NO

Reached value: sum-MStage, time, or sum-MSF (with time unit)

Extrema: relative stiffness, max out-of-balance force, max incremental displacement (each with node ID, DOF, x, y, z)

Optional flags: steady-state and unsaturated status

Phase summary

Outcome message: e.g. "Prescribed ultimate state reached", "Soil body collapses", "Calculation aborted"

THE CHALLENGE

  • Field presence depends on three factors:
  • ① Calculation type
  • Plastic → no flow/thermal fields; THM → all fields; Safety → strength reduction, no time unit
  • ② Element types in model
  • Interface, geogrid, anchor, plate, beam, embedded beam, flex. hinge — absent element = absent fields
  • ③ Runtime solver decisions
  • Rescaling, arc-length, sub-steps, attempts — all conditional
  • → No fixed schema; every field must be detected dynamically
Log File → Header (once) → Step 1 [Iter 1..N → Summary] → Step 2 [...] → ··· → Phase Summary

Methodology — Parsing Logic

Rule-based pipeline — 10 ordered rules, first match wins per line:

① Known block detection & parsing

Detects 6 block headers (Steady-state regime, Check convergence, Flow boundary conditions, Unsaturated behaviour, Freezing/thawing heat storage, Time step upscale validity). Once inside, a state machine per block type handles nested metrics, sub-modes, and YES/NO sentinel lines that close the block.

② Step size

Extracts current step size value with time unit detection (days/seconds/hours/minutes/years) and flags negative steps

③ Total iterations

Parses "Total number of iterations in current step"

④ Plastic points

Detects family headers ("Soil plastic points", "Interface plastic points", etc. — 7 families) and captures Total / Inaccurate / Total tension / Inaccurate tension counts per family.

⑤ Converged attempt

Captures YES/NO from "Converged attempt" line

⑥ Inaccurate soil elastic points

Standalone scalar outside plastic-points families

⑦ Rescaling reason

Forward-pairs a reason line (e.g. "Diverging solution") with the next numeric "Rescaling: step size scale factor" line

⑧ Unknown block handling

Non-indented unrecognised text starts a new unknown block. Subsequent indented lines are captured as nested children using indent-depth hierarchy.

⑨ Standalone YES/NO

Captures isolated status flag lines not inside any block

⑩ Generic trailing metric

Catch-all: any remaining "label value" line is stored as a named metric

Live Tracking — Real-Time Monitoring

  • Monitor a running PLAXIS calculation — parse the convergence log as it grows.
  • Two modes:
  • Companion server (live-server.ts on localhost:9876) — reliable reads of files held open by PLAXIS, supports changing watched directory at runtime
  • File System Access API (browser-only fallback) — no server needed
  • Capabilities:
  • Phase selector modal — scans for data.convd.rr# files, shows phase numbers & sizes
  • Auto-detects newly appearing phases during calculation
  • Per-phase stop / resume — freeze individual phases while tracking others

Summary

PlaxisInsight transforms raw convergence logs into an interactive diagnostic tool.

  • ✓ Parse — PLAXIS calculation types automatically
  • ✓ Visualise — convergence errors, stiffness, plastic points, rescaling — interactive charts
  • ✓ Compare — multiple analyses with overlay charts and summary tables
  • ✓ Custom plots — any metric vs. any metric, step or iteration resolution
  • ✓ Inspect — every iteration — dynamic columns, tolerance colouring, fullscreen mode
  • ✓ Export / Import — lossless JSON round-trip for sharing and archival
  • ⚙ Live tracking — real-time monitoring of running PLAXIS calculations (in development)

Built for the CCG and Support teams — designed for any PLAXIS power user.

Master Prompt for "Methodology — Parsing Logic" Slide
======================================================

Create a presentation slide titled "Methodology — Parsing Logic" for a technical talk about parsing PLAXIS convergence log files. Use the same teal/dark-teal color scheme as other slides. Keep text minimal — this is a presentation slide, not documentation.

---

Layout: Single column, one bold header line followed by a numbered list of rules.

---

Content:

Header line (bold):
Rule-based pipeline — 13 ordered rules, first match wins per line

Each iteration line in the log is passed through these rules in order. The first rule that claims the line processes it; remaining rules are skipped.

Rules (numbered list with circled numbers, short one-liner descriptions):

 1  Start known block — detects 6 block headers: Steady-state regime, Check convergence, Flow boundary conditions, Unsaturated behaviour, Freezing/thawing heat storage, Time step upscale validity

 2  Known-block mode — once inside a known block, a state machine per block type handles nested metrics, sub-modes (e.g. "Check steady state"), and YES/NO sentinel lines that close the block

 3  Unknown-block mode — captures indented lines under unrecognised headers using indent-depth hierarchy; supports nested children

 4  Step size — extracts current step size value with time unit detection (days/seconds/hours/minutes/years) and flags negative steps

 5  Total iterations — parses "Total number of iterations in current step"

 6  Plastic points header — detects family headers like "Soil plastic points", "Interface plastic points" (7 families)

 7  Plastic points rows — inside an active family, captures Total / Inaccurate / Total tension / Inaccurate tension counts

 8  Converged attempt — captures YES/NO from "Converged attempt" line

 9  Inaccurate soil elastic points — standalone scalar outside plastic-points families

10  Rescaling reason — forward-pairs a reason line (e.g. "Diverging solution") with the next numeric "Rescaling: step size scale factor" line

11  Unknown block start — non-indented unrecognised text starts a new unknown block for future indented children

12  Standalone YES/NO — captures isolated status flag lines not inside any block

13  Generic trailing metric — catch-all: any remaining "label  value" line is stored as a named metric
